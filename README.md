# Check It Yourself — A Verification Guide

## What this repository is

The articles you came from make three structural claims: that **two attestations are only as independent as the axes they differ on** (witness count does not simply aggregate); that a **six-step closed flow forces one specific algebra** (the cyclic group of order six — its non-commutative rival of the same size provably cannot carry it); and that **some operations are irreducibly external** to a system's own declared primitives (a "fence": no amount of internal composition reaches them). This repository contains the machine-checked proofs behind those claims, and this guide tells you how to check them yourself, in minutes, without trusting us.

The honest version of the pitch is not "we verified this." It is: **the proof assistant's kernel accepted these proofs, and you can make it re-check them on your own machine.** You are not asked to believe a person. You are asked to run a check.

One thing this repository deliberately is *not*: a second witness for the articles. The articles and these proofs share an author, and one of the theorems in this repository (`shared_axis_strictly_weaker`) is precisely about why that sharing caps the evidence. *Your* build, on *your* machine, is where independence enters.

## The results, in plain language

**1. Witness independence has two axes — and evidence is capped by them** (`WitnessIndependenceProofs/WitnessIndependence.lean`)
An attestor is encoded as a pair of coordinates: which *model* produced the attestation, and which *facet* (kind of property) it certifies. The theorems prove the two axes are genuinely different relations (`axes_not_collinear`), that two attestors sharing either axis constitute strictly weaker evidence than two independent on both (`shared_axis_strictly_weaker`) — witness *count* does not aggregate, axis-independence does — and that an "independence price" built from a scalar coupling is zero exactly when both axes coincide (`priceIndep_eq_zero_iff`). Non-vacuity witnesses are proved in-file, so the inequalities are not vacuously true.

**2. A six-cycle forces ℤ/6ℤ — and S₃ provably can't carry it** (`WitnessIndependenceProofs/HexadOrderSixForcing.lean`)
For any 6-element set with a 6-cycle on it: any order-6 subgroup of its symmetric group containing that cycle *is* the cyclic group generated by it (`order_six_subgroup_containing_six_cycle_is_cyclic`, `…_is_Z6`), and no copy of S₃ — the *other* group of order six, the non-commutative one — can contain a 6-cycle at all (`no_S3_contains_six_cycle`). Two structures of the same size, provably disjoint as carriers of the same data.

**3. The fence: some operators are irreducibly external** (`WitnessIndependenceProofs/SubstrateFenceIrreducibility.lean`)
A system's "internal expansion" is encoded as the composition closure of its declared primitive operators. The abstract theorems are unconditional: an operator not in the closure cannot equal any internally derivable one (`operator_irreducible`), and a non-vacuity gate proves the closure is a proper sub-structure — neither everything nor nothing (`nonvacuity_gate`). Two worked exemplars instantiate this on a 9-element layered cycle whose single declared primitive is the additive successor: the "doubling" flow is provably outside its closure (`hexad_not_derivable`, riding the negative result below). One reading is assumption-free (`w_cross_not_internally_realizable`: a verdict that genuinely depends on external data cannot be computed from internal data alone); the other rests on a declared modelling identification, which the file marks, in capitals, as a choice and not a theorem.

**4. The supporting backbone — including the result that says NO** (`WitnessIndependenceProofs/EnneagramHexadDerivability.lean`, `WitnessIndependenceProofs/BennettPeriodicTableZ9Z.lean`)
The backbone file proves the nine-point structure's two interlocking algebras — the additive 9-cycle and the multiplicative doubling hexad — are genuinely independent: **no iterate of the additive successor equals the doubling map** (`hexad_flow_external_to_additive`). That is a *negative* result, shipped as a first-class theorem: it is what makes the fence exemplars non-circular, and it is the file to read if you suspect a framework that can only say yes. The Bennett file supplies the explicit S₃ group, the proof that S₃ ≇ ℤ/6ℤ (`s3_not_iso_z6z`), and the doubling cycle itself; its closing block/period theorems encode one author's structural reading of the periodic table — whether that reading captures real chemistry is a modelling judgement, flagged in the file header.

## How to check it (10–20 minutes, first run)

You need: git, a terminal, a few GB of disk, and an internet connection.

```bash
# 1. Install elan (the Lean toolchain manager) — one line, official installer:
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# 2. Clone and enter:
git clone https://github.com/field-effect-institute/witness-independence-proofs
cd witness-independence-proofs

# 3. Fetch the prebuilt mathematics library cache:
lake exe cache get

# 4. Build — this is the actual check:
lake build
echo $?
```

If the final line prints `0` after `Build completed successfully`, the Lean 4 kernel — a small, independent core that every proof must pass through, the same one used by mathematicians formalizing research mathematics — has re-verified every theorem in this repository on your hardware. Most of the 10–20 minutes is downloading; the verification itself takes seconds.

The toolchain (`lean-toolchain`) and the mathematics library version (`lake-manifest.json`) are pinned, so you build exactly what we built.

## What a passing build establishes — and what it doesn't

**It establishes:** every stated theorem follows, by machine-checked logical deduction, from the stated definitions. No proof contains a gap, an unproven placeholder (`sorry`), or a hidden assumption beyond Lean and its standard mathematics library. The repository uses no `sorry`, no `admit`, no `axiom` declarations, and none of the faster-but-heavier `native_decide` mechanism (which would extend trust to the compiler) — every check goes through the ordinary kernel path.

**It does not establish:**

- **That the definitions are the world.** The proofs certify the *shape*, not the mind. The fence exemplars encode a cognitive system as a 9-element layered cycle with one declared additive primitive; that encoding is a modelling judgement, and where a file rests on a declared identification rather than a derivation, the file says so explicitly. Anything the articles claim about real cognition — human or machine — is argued there at the register the articles state, not proved here.
- **That anyone independent verified this before you.** The receipts in this repository were produced by the authors' own toolchain runs, and the articles and proofs share an author. *Your* build is the independent verification. That is the point of publishing it.
- **The grander claims.** These files prove (and one *dis*proves) specific algebraic structures. They do not prove any broader framework, methodology, or worldview. Anything in the articles beyond these results is argument, not theorem, and should be weighed as argument.

**One cited result is not in this repository.** The scalar coupling form `1 − |a−b|/R` used in the pricing theorems originates in a separate formalization maintained outside this repository's source tree. The theorems about it here are self-contained — nothing in this repository imports or depends on that external file; it is cited in the articles as the form's origin, not as a dependency of these proofs.

## How to try to break it

- **Change a definition.** Edit `hexad_flow` so doubling wraps differently, rebuild — `hexad_flow_external_to_additive`'s witnesses fail. The proofs are load-bearing on the definitions, not decorative.
- **Hunt for trapdoors.** `grep -rn "sorry\|admit\|axiom" WitnessIndependenceProofs/` — check that what we just said is true.
- **Audit the axioms.** In any file, after a theorem, add `#print axioms <theorem_name>` and rebuild — Lean prints exactly which axioms the proof rests on. (Several theorems here, including `operator_irreducible` and `w_cross_not_internally_realizable`, depend on no axioms at all.)
- **Attack the modeling.** The strongest critique isn't "the proof is wrong" (the kernel settles that); it's "the definition doesn't capture the real thing." Read the definitions and make that case — that's the conversation we're inviting.

## If the build fails — or you break something

A genuine failure on a clean clone with the pinned toolchain would mean the repository does not do what this guide says. Please open an issue with your OS, the command output, and the contents of `lean-toolchain` — that's a real finding, and finding it is what this setup is for.

The same goes for the stronger kind of finding: a case where a definition doesn't capture the real thing it claims to encode. Open an issue, or write to ryan@fieldeffectinstitute.org — accepted refinements go on the record under the name or handle you choose.
